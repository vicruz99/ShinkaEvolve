# B.2. Second autocorrelation inequality
Let $C_2$ be the smallest constant for which one has
$$ \|f*f\|_2^2 \leq C_2 \|f*f\|_1 \|f*f\|_\infty$$
for all non-negative $f:\mathbb{R} \rightarrow \mathbb{R}$.
It is known that
$$ 0.88922 \leq C_2 \leq 1,$$
with the lower bound coming from a step function construction by [Matolcsi and Vinuesa (2010)](https://www.sciencedirect.com/science/article/pii/S0022247X10006001). AlphaEvolve found a step function with 50 equally-spaced intervals on $[-1/4,1/4]$ that gives a slightly better lower bound $0.8962 \leq C_2$.


## Implementation in ShinkaEvolve

This folder contains an implementation of the problem detailed in **Annex B.2** of the AlphaEvolve paper [1] [2]. I highly suggest checking out the [Google DeepMind Colab](https://colab.research.google.com/github/google-deepmind/alphaevolve_results/blob/master/mathematical_results.ipynb#scrollTo=IBOyCNqLeJ2S) for further insights and explanations on how AlphaEvolve approached this problem. Although the third autocorrelation inequality isn't explicitly addressed in this notebook, the other autocorrelation problems are.

The implementation was adapted from open-source code and the notebook [Google DeepMind Colab](https://colab.research.google.com/github/google-deepmind/alphaevolve_results/blob/master/mathematical_results.ipynb#scrollTo=IBOyCNqLeJ2S). All credits go to the authors of the original code.

The files in this repository were adapted for ShinkaEvolve from the following sources:

* **Verification Code:** Adapted from the official [Google DeepMind Colab](https://colab.research.google.com/github/google-deepmind/alphaevolve_results/blob/master/mathematical_results.ipynb#scrollTo=IBOyCNqLeJ2S).
* **Initial Programs:** Four variations are provided (prompts available in `configs/variants`):
    * **Variation 1 (`initial_code_evolve.py`):** Utilizes a random sequence of heights, following the approach in [4]. [CodeEvolve Github](https://github.com/inter-co/science-codeevolve-experiments/blob/main/problems/alphaevolve_math_problems/autocorrelation_problems/third_autocorr_ineq/input/src/init_program.py).
    * **Variation 2 (`initial_alpha_evolve.py`):** Starts with a random sequence but provides the agent with the best-known sequence (`best_sequence_prev_alpha_evolve`) and access to the evaluation function, as suggested in [2].
    * **Variation 3 (`initial_theta_evolve.py`):** Sourced from the **ThetaEvolve** [6] [repository] (https://github.com/ypwang61/ThetaEvolve/tree/main/openevolve_adapted/examples/third_autocorr_inequality).
    * **Variation 4 (`initial_open_evolve.py`):** Sourced from OpenEvolve (originally implemented by Codeevolve). See the [source file](https://github.com/algorithmicsuperintelligence/openevolve/blob/main/examples/alphaevolve_math_problems/third_autocorr_ineq/initial_program.py) and [Pull Request #302](https://github.com/algorithmicsuperintelligence/openevolve/pull/302).

## How to use?

To reproduce these results or run your own evolution experiments, follow these steps:

1.  **Install Dependencies:** Ensure you have the required environment set up by installing the dependencies listed in:
    `examples/alpha_evolve_math_problems/pyproject.toml`
2.  **Configure Variants:** Adjust the evolution parameters or model settings by editing the variant configuration files located here:
    `configs/variant/alpha_evolve_math_problems/third_autocorr_ineq`
3.  **(Optional) Visualize Results:** After running the evolution, you can analyze the generated step functions and $C_3$ convergence using the provided notebook:
    `examples/alpha_evolve_math_problems/third_autocorr_ineq/visualization.ipynb`
